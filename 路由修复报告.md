## 🔍 问题分析

### 问题1: knowledge_point_stats表不存在

**错误日志**:
```
SqliteError: no such table: knowledge_point_stats
    at Database.prepare
    at routes/knowledge-points.js:40:22
```

**根本原因**:
- 迁移脚本 `migrate_knowledge_points.js` 未执行
- 数据库中没有knowledge_point_stats表

**影响范围**:
- 知识点统计功能
- 薄弱知识点识别功能
- 学习数据仪表盘

---

### 问题2: skills表结构不兼容

**错误日志**:
```
SqliteError: no such column: required_level
    at Database.prepare
    at routes/skills.js:15:26
```

**根本原因**:
- 旧的skills表存在(用于战斗技能)
- 新的skills表结构不同(用于宠物技能系统)
- 迁移脚本使用 `CREATE TABLE IF NOT EXISTS`,没有更新旧表

**旧表结构**:
```sql
skills (
  id, name, type, power, accuracy, element, description, animation_id, cooldown
)
```

**新表结构**:
```sql
skills (
  id, name, description, icon, skill_type, subject,
  base_damage, base_defense, base_speed, cooldown,
  required_level, required_knowledge_point, required_accuracy,
  created_at
)
```

**影响范围**:
- 宠物技能系统
- 技能学习/升级功能
- 知识点关联技能

### 修复2: 更新skills表迁移脚本

**修改文件**: `backend/scripts/migrate_pet_skills.js`

**修改内容**:
```javascript
// 添加旧表检测和删除逻辑
const oldTableExists = db.prepare(
  "SELECT name FROM sqlite_master WHERE type='table' AND name='skills'"
).get();

if (oldTableExists) {
  console.log('⚠️  检测到旧版skills表,正在删除...');
  db.prepare('DROP TABLE IF EXISTS skills').run();
  db.prepare('DROP TABLE IF EXISTS pet_skills').run();
}

// 使用 CREATE TABLE 而不是 CREATE TABLE IF NOT EXISTS
db.prepare(`
  CREATE TABLE skills (
    -- 新表结构
  )
`).run();
```

**执行命令**:
