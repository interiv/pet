## 系统概述

班级宠物养成系统是一个创新的教育游戏化平台,将宠物养成与学习结合,通过游戏化机制提升学生学习动力和班级凝聚力。

### 核心理念
- **学习即游戏**: 完成作业=获得经验和金币
- **知识即力量**: 掌握知识点=解锁宠物技能
- **团队即力量**: 班级协作=击败学习BOSS

### 技术栈
- **前端**: React 18 + TypeScript + Ant Design + Vite
- **后端**: Node.js + Express + SQLite
- **认证**: JWT Token
- **实时通信**: Socket.IO

## 快速开始

### 环境要求
- Node.js >= 16.0
- npm >= 8.0
- 现代浏览器(Chrome/Firefox/Safari/Edge)

### 1. 克隆项目
```bash
git clone <repository-url>
cd pet
```

### 2. 安装依赖

#### 后端
```bash
cd backend
npm install
```

#### 前端
```bash
cd frontend
npm install
```

### 3. 配置环境变量

创建 `backend/.env`:
```env
PORT=3000
JWT_SECRET=your-secret-key-change-this
FRONTEND_URL=http://localhost:5173
DATABASE_PATH=./data/database.sqlite
```

创建 `frontend/.env`:
```env
VITE_API_URL=http://localhost:3000/api
```

### 4. 数据库迁移

**必须按顺序执行**:

```bash
cd backend

# 1. 初始化基础数据库
node scripts/init_db.js

# 2. 每日任务系统
node scripts/migrate_daily_tasks.js

# 3. 知识点统计
node scripts/migrate_knowledge_points.js

# 4. 宠物技能系统
node scripts/migrate_pet_skills.js

# 5. 班级BOSS战
node scripts/migrate_boss_battles.js
```

### 5. 启动服务

#### 开发模式

后端:
```bash
cd backend
npm run dev
```

前端:
```bash
cd frontend
npm run dev
```

访问: http://localhost:5173

#### 生产模式

后端:
```bash
cd backend
npm start
```

前端:
```bash
cd frontend
npm run build
# 将dist目录部署到静态服务器
```

## 数据库迁移

### 迁移脚本说明

| 脚本 | 功能 | 新增表 |
|------|------|--------|
| `init_db.js` | 初始化基础数据库 | users, pets, assignments等 |
| `migrate_daily_tasks.js` | 每日任务系统 | daily_tasks, daily_task_logs |
| `migrate_knowledge_points.js` | 知识点统计 | knowledge_point_stats |
| `migrate_pet_skills.js` | 宠物技能 | skills, pet_skills |
| `migrate_boss_battles.js` | 班级BOSS战 | boss_battles, boss_battle_participants, boss_battle_rewards |

### 数据库结构

```
database.sqlite
├── 用户系统
│   ├── users (用户表)
│   └── classes (班级表)
├── 宠物系统
│   ├── pets (宠物表)
│   ├── pet_species (宠物种类)
│   ├── pet_skills (宠物技能)
│   └── skills (技能定义)
├── 学习系统
│   ├── assignments (作业表)
│   ├── question_bank (题库)
│   ├── submissions (提交记录)
│   ├── wrong_questions (错题本)
│   └── knowledge_point_stats (知识点统计)
├── 游戏系统
│   ├── items (道具)
│   ├── user_items (用户道具)
│   ├── achievements (成就)
│   ├── battles (战斗记录)
│   └── boss_battles (BOSS战)
└── 任务系统
    ├── daily_tasks (每日任务)
    └── daily_task_logs (任务日志)
```

## 部署指南

### 方案一：单机部署 (推荐测试/小规模)

#### 1. 准备服务器
- Ubuntu 20.04+ / CentOS 8+
- 2GB RAM 最低
- 10GB 磁盘空间

#### 2. 安装Node.js
```bash
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt-get install -y nodejs
```

#### 3. 部署后端
```bash
# 创建应用目录
mkdir -p /var/www/pet-system
cd /var/www/pet-system

# 上传后端代码
# ... (使用SCP或Git)

# 安装依赖
cd backend
npm install --production

# 运行迁移
node scripts/init_db.js
node scripts/migrate_daily_tasks.js
node scripts/migrate_knowledge_points.js
node scripts/migrate_pet_skills.js
node scripts/migrate_boss_battles.js

# 使用PM2管理进程
npm install -g pm2
pm2 start src/server.js --name pet-backend
pm2 save
pm2 startup
```

#### 4. 部署前端
```bash
cd frontend

# 安装依赖
npm install

# 构建
npm run build

# 使用Nginx服务
sudo apt install nginx
sudo cp nginx.conf /etc/nginx/sites-available/pet-system
sudo ln -s /etc/nginx/sites-available/pet-system /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

Nginx配置示例:
```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前端静态文件
    location / {
        root /var/www/pet-system/frontend/dist;
        try_files $uri $uri/ /index.html;
    }

    # 后端API代理
    location /api {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }

    # WebSocket
    location /socket.io {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
    }
}
```

### 方案二：Docker部署 (推荐生产环境)

#### 1. 创建Dockerfile

后端 `backend/Dockerfile`:
```dockerfile
FROM node:18-alpine
WORKDIR /app
COPY package*.json ./
RUN npm install --production
COPY . .
EXPOSE 3000
CMD ["node", "src/server.js"]
```

前端 `frontend/Dockerfile`:
```dockerfile
FROM node:18-alpine as build
WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
RUN npm run build

FROM nginx:alpine
COPY --from=build /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

#### 2. 创建docker-compose.yml

```yaml
version: '3.8'

services:
  backend:
    build: ./backend
    ports:
      - "3000:3000"
    environment:
      - NODE_ENV=production
      - JWT_SECRET=your-secret-key
      - DATABASE_PATH=/data/database.sqlite
    volumes:
      - ./backend/data:/data
    restart: unless-stopped

  frontend:
    build: ./frontend
    ports:
      - "80:80"
    depends_on:
      - backend
    restart: unless-stopped
```

#### 3. 启动
```bash
docker-compose up -d
```

## 用户手册

### 学生用户

#### 1. 注册与登录
- 访问系统,点击"注册"
- 填写用户名、密码、邮箱
- 选择班级(或输入邀请码)
- 登录后开始新手引导

#### 2. 创建宠物
- 从32种宠物中选择
- 查看进化形态预览
- 设置宠物名称
- 开始养成之旅

#### 3. 完成作业
- 点击侧边栏"班级作业"
- 查看待完成作业
- 答题并提交
- 获得经验、金币、宠物心情提升

#### 4. 投喂宠物
- 点击"我的宠物"
- 使用道具投喂
- 提升饱腹度/心情/属性
- 自动更新每日任务

#### 5. 参与BOSS战
- 点击"BOSS战"菜单
- 查看班级当前BOSS
- 点击"攻击BOSS"(通过答题)
- 查看伤害排行榜
- BOSS被击败后领取奖励

#### 6. 学习技能
- 点击"宠物技能"
- 查看可学习技能
- 满足条件后学习
- 消耗金币升级技能

#### 7. 查看学习数据
- 点击"学习数据"
- 查看知识点掌握情况
- 关注薄弱知识点
- 根据建议复习

### 教师用户

#### 1. 创建班级
- 登录后进入"班级管理"
- 点击"创建班级"
- 设置班级名称、描述
- 获取邀请码

#### 2. 发布作业
- 点击"发布新作业"
- 填写科目、主题、难度
- 选择题型和数量
- AI生成题目

#### 3. 精修题目
- 预览AI生成的题目
- 点击"编辑"修改题目
- 调整难度和知识点
- 删除不合适的题目
- 确认发布

#### 4. 创建BOSS
- 手动创建: 填写BOSS信息
- 自动生成: 从班级错题生成
- 设置BOSS等级和持续时间
- 学生通过答题攻击BOSS

#### 5. 查看统计
- 查看班级作业完成情况
- 查看学生成绩排行
- 查看知识点掌握情况
- 下载学习报告

  "username": "string",

  "email": "string",

#### POST /api/auth/login

  "username": "string",
  "password": "string"

  "species_id": "number",

## 故障排除

### 常见问题

#### 1. 数据库迁移失败
**问题**: 执行迁移脚本报错
**解决**:
```bash
# 删除数据库重新初始化
rm backend/data/database.sqlite
node scripts/init_db.js
# 重新执行所有迁移
```

#### 2. 前端无法连接后端
**问题**: API请求404或CORS错误
**解决**:
- 检查`.env`文件中的API_URL
- 确认后端服务已启动
- 检查CORS配置

#### 3. JWT认证失败
**问题**: 登录后请求返回401
**解决**:
- 清除浏览器localStorage
- 重新登录
- 检查JWT_SECRET配置

#### 4. 图片无法显示
**问题**: 宠物图片404
**解决**:
- 检查`frontend/public/images`目录
- 确认图片路径正确
- 检查file://协议问题

#### 5. BOSS战无法创建
**问题**: 创建BOSS报错
**解决**:
- 确认已执行boss_battles迁移
- 检查班级ID是否正确
- 确认教师权限
